# Handoff Notes вЂ” 2026-04-07

## Project Overview

ClaudeClaw (`assistant-runtime`) is a Python personal-assistant runtime that acts as a backend router between messaging channels (Telegram, Discord, Slack) and the local `claude` CLI. It is a spiritual port of "OpenClaw" (a large TypeScript monorepo) adapted for Claude Code as the AI backend. ~9,200+ lines of Python, **55 tests, all passing**.

See `docs/what-is-claudeclaw.md` for a plain-English explanation of the project. See `docs/roadmap.md` for strategic direction and remaining work.

**Implementation sketch:**

- Router: prepend extracted text to prompt with a header line

## Key Architecture Notes

### Message flow
1. `AssistantRouter.run()` starts one polling thread per account
2. `channel.get_updates()` в†’ `ChannelMessage` objects
3. `_handle_message_locked()` вЂ” per-chat lock:
   - Auth check в†’ slash command check в†’ cache/cooldown в†’ context load в†’ `_generate_reply_with_tools()`

### Two execution paths in `_generate_reply_with_tools`
- **Streaming** (Telegram): sends `в–Њ` placeholder, edits in place as tokens arrive, throttled 300ms, tool calls show status lines
- **Blocking** (Discord, Slack, fallback): `subprocess.run()`, full response, up to 3 tool iterations

### BriefingThread pattern (new)
Modeled on `ConsolidationThread` in `memory.py`. Wakes hourly via `threading.Event.wait(3600)`, fires once per `(YYYY-MM-DD:HH)` slot. Register senders with `register_sender(key, callback)` and targets with `register_target(key, chat_id)` вЂ” same pattern as `Scheduler`.

### `assistant add` channel wiring
Uses `load_raw_config()` + `update_config_values()` with a read-modify-write pattern to add a new account without clobbering existing ones. Key: shallow merge only works at the top level, so accounts dict must be loaded first, modified, then written back as a whole.

## Privacy / Safety Note

ClaudeClaw's key differentiator vs OpenClaw: **everything runs locally**. Conversation history, memory, config, and tool execution never leave the machine. The only external data flow is the message в†’ Anthropic API call (same as Claude.ai). No ClaudeClaw server, no third-party backend, no sync service. See `docs/what-is-claudeclaw.md` for the full explanation.
