## Known Remaining Issues / Open Questions

### 1. Windows installed lifecycle is improved but not perfectly clean
The biggest remaining rough area is Windows lifecycle behavior in real installed mode.

What is known:
- the runtime can launch
- the runtime log shows the app entering the polling loop
- fake Telegram token causes repeated Telegram 404 errors in the log (expected with test token)
- status now falls back to lock PID evidence and is more honest than before

What is still not fully resolved:
- runtime-owned PID confirmation is still flaky in installed Windows launch behavior
- the CLI currently relies on pragmatic evidence rather than a perfectly clean PID handshake

### 2. Stale/legacy runtime state can still muddy debugging
The lifecycle logic was improved significantly, but future cleanup work may still be needed for a cleaner Windows process ownership model.

### 3. Telegram 404 spam with invalid token
This is expected with a fake token, but operator messaging could be improved later so bad-token cases are easier to diagnose without log spam.

## Late-Night Follow-up: ClaudeClaw planning + responsiveness work

### Architecture/docs
- added `docs/claudeclaw-architecture.md`
- updated `docs/roadmap.md` so the next practical priorities now point at:
  - parallel per-account polling
  - latency instrumentation
  - memory retrieval
  - first real tool substrate

### Runtime responsiveness refactor
- refactored `app/router.py` so Telegram polling is no longer a single sequential long-poll loop over every configured account
- runtime now starts **one worker thread per account**, each independently long-polling `getUpdates`
- added worker monitoring so the main thread exits loudly if an account worker dies unexpectedly
- this should remove the main architectural latency smell from the first multi-bot milestone, where one quiet account could delay another account's responsiveness by occupying the sequential polling loop

### Test updates
- expanded `tests/test_routing.py` to cover:
  - one worker thread per account
  - surfaced worker failures
  - dead worker detection
