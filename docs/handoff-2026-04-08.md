# Handoff Notes — 2026-04-08

## Project Overview

ClaudeClaw (`assistant-runtime`) is a Python personal-assistant runtime that routes messages between messaging channels (Telegram, Discord, Slack) and the local `claude` CLI. It's a spiritual port of "OpenClaw" adapted for Claude Code as the AI backend. ~21,500 lines of Python, **55 tests, all passing**.

Default install path: `~/ClaudeClaw/assistant` (Mac/Linux) / `%USERPROFILE%\ClaudeClaw\assistant` (Windows)

See `GUIDE.md` for full user documentation. See `docs/roadmap.md` for strategic direction.

## Key Architecture Notes

### Message flow
1. `AssistantRouter.run()` starts one polling thread per account
2. `channel.get_updates()` → `ChannelMessage` objects
3. `_handle_message_locked()` — per-chat lock:
   - Auth check (or DM pairing) → slash command check → cache/cooldown → compaction → context load → `_generate_reply_with_tools()`

### Two execution paths in `_generate_reply_with_tools`
- **Streaming** (Telegram): sends `▌` placeholder, edits in place as tokens arrive, throttled 300ms, tool calls show status lines
- **Blocking** (Discord, Slack, fallback): `subprocess.run()`, full response, up to 3 tool iterations

### Compaction flow
1. Before reading transcript, `SessionCompactor.maybe_compact()` checks token budget
2. If over threshold: reads transcript, splits 60/40, calls Claude to summarize older portion
3. Writes compaction marker (`direction="compaction"`) to JSONL — append-only, never mutates
4. `read_transcript_with_compaction()` finds last marker, returns `(summary, recent_entries)`
5. `build_prompt()` injects summary section before recent transcript

### Claude CLI invocation
- Prompt passed via stdin (`-p -`) to avoid Windows command-line length limits
- Both `subprocess.run()` (blocking) and `subprocess.Popen` (streaming) use this pattern

### Response cache
- Key: `(chat_id, agent_name, normalized_message)`
- TTL: configurable (default 300s)
- Max entries: 500 with oldest-first eviction
- Expired entries cleaned lazily on access + batch on set
